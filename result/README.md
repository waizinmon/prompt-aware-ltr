## Data files (JSON)

These are produced on Colab (GPU) or Mac (CPU) and should be placed here:

- `results_dataset1.json` / `results_dataset2.json` — the frozen, reported latency-vs-request-rate benchmark sweeps (Dataset 1 in-distribution, Dataset 2 out-of-distribution), produced by `scripts/3_vllm_benchmark.py`. These are the numbers in the report — do not overwrite.
- `ttft_dataset1.json` / `ttft_dataset2.json` — Time-to-First-Token sweeps (same schedulers/rates as above), produced by `scripts/3_vllm_benchmark.py`'s TTFT tracking. Kept as separate files from `results_dataset*.json` so the frozen latency numbers are never touched by a rerun.
- `extended_metrics.json` — throughput (and optionally live GPU memory/utilization) computed from `results_dataset*.json`, produced by `scripts/5_extended_metrics.py`.
- `hol_blocking_metrics.json` — Head-of-Line blocking indicators (p90/mean latency ratio at high load, latency-vs-rate slope) computed from `results_dataset*.json`, produced by `scripts/6_hol_blocking_eval.py`.
- `tau_results.json` — Kendall's Tau ranking-quality comparison (Original LTR vs. LTR + Prompt Length), produced by `scripts/2_kendalls_tau_eval.py`.
